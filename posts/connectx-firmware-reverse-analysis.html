从固件二进制里挖出 NVIDIA 的未公开网卡

2026-06-19

摘要：本文对 NVIDIA ConnectX-7/8/9 三大家族的官方固件下载页面进行全量 API 枚举（50+13+2 版本），下载代表性固件做二进制扫描，并与 NVIDIA 官方用户手册（PDF 解析）、开发者论坛、GitHub 代码、30+ 家 OEM 销售页面、供应链分析进行交叉验证。发现了 7 个固件中有但公开文档未确认的型号，确认了 2 个 NVD 前缀内部 PSID 的真实身份，以及 DGX GB300 系列网卡的完整对应关系。

一、方法：全量枚举 + 固件扫描 + 手册交叉

NVIDIA 的固件下载页面不是静态 HTML，而是用一个 Azure 动态加载器 downloaders.azurewebsites.net 通过几轮 POST 请求来填充内容。我写了套 Python 脚本，通过以下步骤完成全量枚举：

  1. 获取版本列表：POST .../helper.php action=get_versions → 返回所有版本、GA 版本、最新版本
  2. 获取每个版本的 OPN：action=get_distros&version=xx → 返回该版本支持的所有 OPN
  3. 获取 PSID：action=get_oses&version=xx&distro=opn → 返回的数组元素就是 PSID（如 MT_0000001167）
  4. 获取下载链接：action=get_download_info → 返回固件 ZIP 的 URL 和校验值

枚举完成后，下载了 25 个代表性固件样本（覆盖每个家族的最新 GA、最老版本、首次出现新 OPN 的版本、LTS 版本）。固件是 .signed.bin 格式，64MB，带签名。标准的 FS4 MTFW / ITOC / DTOC 结构在 .signed.bin 中不在标准偏移位置，但 PSID 作为 ASCII 字符串保留在固件数据中，通过全量正则扫描可以可靠提取。这是核心策略——与 mstflint 的 -no_dev 签名镜像处理逻辑一致。

最后，拿着发现的 OPN/PSID 列表，与以下来源进行交叉验证：

  • NVIDIA 官方 ConnectX-8/9 用户手册（PDF 解析，132 页 + 109 页）
  • NVIDIA Developer Forums（forums.developer.nvidia.com）
  • GitHub 代码仓库（NVIDIA 官方设备树、Mellanox 工具源码）
  • 30+ 家 OEM 销售页面（MSI、ASUS、Exxact、Supermicro、Lenovo 等）
  • 供应链分析（SemiAnalysis、TradeLoop）
  • 现场拆解照片（ServeTheHome GTC 2025）
  • 固件下载站（Softpedia、Ebay、Volza 海关数据）

4.2 关键时间线

40.44.0208 (最早) → C8180 (ST0) + C8240 (CN-ST0) [量产首发] 40.45.1020 → + C8180 DSP (DT0) [量产扩展] 40.46.1006 [里程碑] → + C8180-NX (ST0) + C8180-NX-DSP (DT0) + C8180P-MC (MC0) + C8180P-NX-MC (MC0) + C8240-SN (ST0) [5个新OPN同时出现] 40.47.1026 → + P6612 (NVD0000000072) [内部开发板] 40.48.1000 → + C8220 (CV-ST0) [200G Prototype] 40.49.1014 (最新) → 支持全部 10 个固件 OPN 手册 2025-05 → 新增 C8180L (SL0), C8180P MP0/MJ0 [固件未确认] 手册 2025-12 → 新增 C8220 (CV-ST0) [固件已确认]

结论：没有任何原始版本 (40.44.x) 支持未发售卡。手册确认但固件未出现的 OPN (C8180L/MP0/MJ0) 可能在后续固件中加入。

六、PSID 编号规律

家族PSID 范围示例
CX7MT_00000007xx ~ 11xx0784, 0834, 0844, 1121
CX8MT_00000011xx ~ 17xx1166, 1222, 1305, 1763
CX9MT_00000015xx ~ 17xx1589, 1715

PSID 编号是跨家族连续分配的，而非每个家族独立编号。

内部/合作伙伴 PSID（NVD 前缀）跨家族连续递增：

NVD0000000033 (P3740, Jetson/IGX) → NVD0000000054 (GB200 Mezzanine) → NVD0000000072 (P6612, DGX-B200)

八、系统级对应关系

8.1 P3740-B0-QSFP / NVD0000000033 — Jetson IGX Orin/Thor 底板

最终确认：P3740 不是芯片，而是 IGX 系列底板的 PCB 编号。

属性说明
P3740 身份IGX Orin / IGX Thor 的底板（Baseboard）PCB 编号
实际芯片NVIDIA ConnectX-7（SmartNIC + 集成 32-lane Gen5 PCIe Switch）
网络端口2x QSFP28（Orin: 100GbE/口；Thor: 200GbE/口）
PCIe 拓扑Tegra SoC → x8 Gen5 → ConnectX-7 → [x16 dGPU] + [x8 PCIe] + [2x QSFP28]
安全 MCUInfineon Aurix TC397（固件：P3740-Aurix-StepB_C01.hex）
设备树tegra234-p3740-0002+p3701-0008.dts
PSIDNVD0000000033

确认来源：NVIDIA Developer Forums (KevinFFF 官方确认)、Linux 内核主线、Holoscan SDK、MLNX_OFED、PNY/ASRock/Aetina。

8.2 900-24768-0002 / NVD0000000054 — GB200 NVL 系统 Mezzanine 板

GB200 NVL 系统的 Dual ConnectX-7 Mezzanine 夹层板，通过 Mirror Mezz 连接器安装在 Bianca 主板上，与 CPU/GPU 共享液冷。固件活跃：28.43.1014 → 28.46.3048 → 28.47.1088 LTS。确认来源：ASUS 产品页、Supermicro GB200 手册、SemiAnalysis 供应链分析。

九、安全性与兼容性警告
