从固件二进制里挖出 NVIDIA 的"未公开网卡"

2026-06-19

最近在整理一套 ConnectX 固件逆向工具，顺手对 NVIDIA 官方固件下载页面做了全量 API 枚举。结果挺有意思——发现了 7 个固件里有、但公开文档里找不到或从未发售的型号，还顺手把 P3740、P6612 这些神秘 OPN 的"真实身份"给确认了。

一、怎么做的：全量枚举 + 固件扫描 + 手册交叉

NVIDIA 的固件下载页面不是静态的，背后有个 Azure 动态加载器 downloaders.azurewebsites.net，通过几轮 POST 可以拿到全部版本、OPN 和下载链接。我写了套脚本，把 CX7（50 版本）、CX8（13 版本）、CX9（2 版本）全量跑了一遍，顺手下载了代表性样本做二进制扫描。

固件是 .signed.bin 格式，64MB，带签名。ITOC 偏移被签名搞乱了，但 PSID 作为 ASCII 字符串还躺在里面，靠全量扫描就能提取。然后拿着这些 PSID 和 OPN，跟 NVIDIA 官方用户手册（PDF 解析）、开发者论坛、GitHub 代码、30 多家 OEM 销售页面、供应链分析做了交叉验证。

二、CX8 发现：7 个未发售/未公开型号

ConnectX-8 是 NVIDIA 最新的 800G 网卡家族，目前只有 13 个固件版本，但已经冒出了 10 个不同 OPN。其中 3 个量产，7 个在公开文档中要么没影、要么标记为原型：

三、CX7 发现：两个 NVD 前缀的"神秘 OPN"

ConnectX-7 有 71 个 OPN，其中两个用 NVD 前缀 PSID，是内部/系统级设备：

  • P3740-B0-QSFP（NVD0000000033）—— 不是芯片，是 IGX Orin/Thor 的底板编号。底板上焊接的是 ConnectX-7，提供 2x QSFP28 + 32-lane Gen5 PCIe Switch。开发者论坛（KevinFFF 官方确认）、Linux 内核设备树、Holoscan SDK 都点过名。
  • 900-24768-0002（NVD0000000054）—— GB200 NVL 系统的 Dual ConnectX-7 Mezzanine 夹层板，通过 Mirror Mezz 连接器安装在 Bianca 主板上，共享液冷。ASUS 产品页、Supermicro 手册、SemiAnalysis 供应链分析都确认过。

四、CX9 与系统级对应

ConnectX-9 目前只有 2 个版本，手册（2025-05 首版）确认了型号是 C9180（不是之前猜的 C9280）和 C9240，目前状态为 Prototype。值得注意的是，固件中的 PSID 编号（MT_0000001589/1715）与 CX8 是连续分配的，说明跨家族用的是同一条编号池。

五、DGX GB300 的网卡到底是谁？

这是全量搜索中最有趣的部分。我翻了 30 多家 OEM 的 GB300 Station 销售页面、ServeTheHome 的现场拆解、NVIDIA 开发者论坛，结论非常一致：

  • DGX Station GB300 Workstation：主板集成的 C8240（900-9X81Q-00CN-ST0），2x QSFP112，400GbE/口。所有 OEM（MSI、ASUS、Exxact、Bizon-Tech 等）规格表一致。
  • DGX GB300 NVL72 Rack Server：用的是独立的 900-9X86E-00CX-SP0/ST0 CX-8 I/O Board，4x OSFP，800Gb/s/口。这个 OPN 没出现在当前固件下载页面，可能通过 DGX OS 系统镜像单独分发。

六、一些有趣的规律

  • PSID 编号跨家族连续：CX7 覆盖 700-1100，CX8 覆盖 1100-1700，CX9 从 1589 开始，说明不是每个家族独立编号。
  • NVD 前缀 PSID 递增：0033（P3740）→ 0054（GB200 Mezzanine）→ 0072（P6612），内部设备按顺序分配。
  • NX 后缀 = Crypto Disabled，手册直接确认。SN 后缀 = Single-port，零售商 T.E.S 的产品页面确认。
  • 没有任何原始版本固件支持未发售卡——它们全部在 40.46.1006 或之后才出现。
