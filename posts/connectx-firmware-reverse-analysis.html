从固件二进制里挖出 NVIDIA 的未公开网卡

2026-06-19 · ConnectX-7 / 8 / 9 三大家族全量逆向

摘要：逐版本解析 NVIDIA ConnectX-7/8/9 三大家族的官方固件 Release Notes（CX7 49 版、CX8 13 版、CX9 2 版的器件表 + 新特性 + Bug 修复 + 已知问题 + 软件版本），结合固件下载器后端 API 枚举、固件二进制 PSID 扫描、官方硬件手册与全网公开信息交叉验证。结论包括：三家族共用同一套固件代码库；NVIDIA 在 同一个 .46 build 上跨家族同步撤掉了 RN 器件表；CX7 完整列出 37 个 PSID（含 GH100「Freysa」内部板）；CX8 有 6 个未发售/内部 SKU；并理清了 IGX、GB200、GB300、DGX-B200 各整机的网卡对应。

核心结论（跨家族 10 条）

  1. 三家族同一套固件代码库：CX7 28.49.1014 / CX8 40.49.1014 / CX9 82.49.1014 的新特性逐条相同（PCC/ZTR-RTT 拥塞直方图、PSP transport 包重整、持久化 CRDT token）。主版本前缀 28/40/82 仅标识硅片，build 号与功能集共享。
  2. .46 是全家族文档分水岭：CX7 在 28.46.1006、CX8 在 40.46.1006 同时删除了 RN 器件表（改为"见硬件文档"）。同一日历期、同一 .46 build 的策略变更。
  3. 披露完整度逐代收紧：CX7 的 RN 器件表曾列多达 28 个 PSID；CX8 只列 3 个旗舰；CX9 从未有过器件表。
  4. PSID 是全公司连续编号池：CX7≈784–1244、CX8=1166–1763、CX9=1589/1715（落在 CX8 区间内）。NVD 内部前缀同样连续：0023→0024→0029→0033→0054→0072。
  5. 未发售卡都是中后期才进固件：三家族原始版（CX7 28.33.2028 / CX8 40.44.0208 / CX9 82.48.1000）均只含量产旗舰。
  6. 工程样品会被回收：CX7 的 MT_0000000856（MCX715105AS-WEA）只活了 28.39→28.41，被量产版 1244 取代；CX8 的无加密 NX 系列只活到 40.47.2682、40.48 起消失。
  7. NVD 前缀 = 系统级嵌入式板：CX7 的 P3740(IGX 底板)、900-24768-0002(GB200 Mezzanine)、Freysa P1011(GH100+CX7)；CX8 的 P6612(DGX-B200 主板)。
  8. 签名固件靠明文 PSID 扫描：签名固件结构表（ITOC）因加密不可遍历，PSID 以 ASCII 明文残留——这是逆向的唯一可靠路径。
  9. 固件尺寸：CX7 标准卡 32MB、OCP/系统级 64MB（唯一两种尺寸的家族）；CX8/CX9 全部 64MB。
  10. GB300 用两种 CX8：DGX Station GB300 = 板载 C8240；GB300 NVL72 = 独立 4×OSFP I/O Board（900-9X86E-00CX-*，不在公开固件页）。

附录 · 枚举方法

固件下载页型号表由动态下载器加载，静态抓不到，但后端可枚举：

POST https://downloaders.azurewebsites.net/downloaders/<family>_downloader/helper.php action=get_versions / get_distros&version=<ver> / get_oses&version=<ver>&distro=<opn> # family ∈ {connectx7, connectx8, connectx9}；get_oses 返回的数组元素即 PSID

各版本 Release Notes（器件表 + 新特性/修复/已知问题/软件版本）为服务端渲染静态页，可直接解析：
https://networking-docs.nvidia.com/connectx<N>fwrn/<version>/<page>，
<page> ∈ {firmware-compatible-products, changes-and-new-features, bug-fixes-in-this-firmware-version, known-issues}。
